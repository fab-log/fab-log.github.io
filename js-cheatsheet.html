Funktionen

Allgemeine Schreibweise

function sum(a, b) {
    return a + b;
}

 

Funktionen können als Variablen definiert und später aufgerufen werden werden.

​x
const sum = function(a, b) {
    return a + b;
};


// ...


sum(x, y);

 

Soll die Funktion nicht sofort ausgeführt, sondern nur auf sie verwiesen werden, darf keine Klammer nach der Funktion stehen.

xxxxxxxxxx
const sayHello = function() {
    console.log("Hello World!");
}


// ...


sayHello();     // wird sofort ausgeführt


// ...


const button = document.querySelector("button");
button.addEventListener("click", sayHello);     // wird ausgeführt, wenn der button geklickt wird

 

 

if

xxxxxxxxxx
const grade = 10;


if (grade > 10) {
    console.log("Passing grade");
} else if (grade === 10) {
    console.log("Passing on the limit");
} else {
    console.log("Failing grade");
}

 

Für booleans genügt die kürzere Schreibweise, da sowieso true oder false zurückgegeben werden.

xxxxxxxxxx
function isPassing(grade) {
    return grade >= 10;
}

 

switch

switch kann alternativ zu if-statements genutzt werden. case gibt dabei die zu prüfende Bedingung an. break beendet die Prüfung, falls case zutrifft. default entspricht dem else-statement und wird ausgeführt, falls keine Prüfung erfolgreich war.

xxxxxxxxxx
const fruit = 'watermelon';


switch (fruit) {
  case 'Mandarins':
    console.log('Mandarins are $2.59 a pound.');
    break;
  case 'cantaloupes':
    console.log('cantaloupes are $5.29 a pound.');
    break;
  case 'watermelon':
    console.log('cantaloupes are $3.79 a pound.'); // expected output
    break;
  default:
    // If not cases are matched, then execute the following code
    console.log(`Sorry, we are out of ${fruit}.`);
}

 

Ternary operator (ternary chain)

xxxxxxxxxx
let num = 1;
let word;
if (num === 1) {
    word = 'one'
} else if (num === 2) {
    word = 'two'
} else if (num === 3) {
    word = 'three'
} else {
    num = 'unknown'
};

kann umgeschrieben werden in:

xxxxxxxxxx
let num = 1;
let word =
  num === 1
    ? 'one'
    : num === 2
    ? 'two'
    : num === 3
    ? 'three'
    : 'unknown';

 

for

xxxxxxxxxx
for (let counter = 1; counter <= 10; counter++) {
    console.log(counter); // 1, 2, 3, ... 10
}


// ######


for (let counter = 10; counter > 0; counter--) {
    console.log(counter); // 10, 9, 8, ... 1
}


// ######


for (let counter = 1; counter <= 10; counter += 2) {
    console.log(counter); // 1, 3, 5, 7, 9
}

In der Klammer nach for werden drei Definitionen vorgegeben:

  1. Der Ausgangswert der Variable zu Beginn der Schleife

  2. Eine Bedingung, die erfüllt sein muss damit die Schleife weiterläuft (und damit das Ende der Schleife festlegt)

  3. Die Operation, die bei jedem Durchlaufen der Schleife vorgenommen werden soll

Achtung! Semikolon zwischen den einzelnen Definitionen

 

for ... of

Anwendung nur für Arrays!

xxxxxxxxxx
const people = ["Sam", "Alex"];


for (const person of people) {
    console.log(person);
}

liefert dieselben Ergebnisse wie

xxxxxxxxxx
people.forEach(element => {
    console.log(element);
});

 

for ... in

Anwendung nur für Objects!

xxxxxxxxxx
const person = {
    id: 1,
    name: "Alex"
};


for (const key in person) {
    console.log(key);
    console.log(person[key]);
}

 

Arrow function

xxxxxxxxxx
const sum = (a, b) => {
    return a + b;
}

 

lexical scope Wann immer this benötigt wird (Klassen!), sollte ausschließlich die arrow-function-Schreibweise genutzt werden, da so die mit this definierte Variable im inneren und im äußeren Bereich der Funktion gleichermaßen genutzt werden kann (behält den zugeordneten Wert bei).

 

Implicit return

xxxxxxxxxx
const sum = (a, b) => a + b;

Voraussetzungen:

  1. Muss eine arrow function sein

  2. darf nur eine Zeile beinhalten

  3. return darf nicht mehr ausgeschrieben werden

 

try ... catch

Falls man den Verdacht hat, dass eine Funktion fehlerhaft/nicht vorhanden sein könnte, kann man sie mit folgender Schreibweise versuchen auszuführen und im Falle eines Scheiterns umgehen.

xxxxxxxxxx
try {
    nonExistentFunction(); // eventuell fehlerhafte Funktion
} catch (error) {
    console.error(error); // interne Fehlermeldung in der Konsole
    return "Ein Fehler ist aufgetreten"; // Fehlermeldung für den Anwender. (Alternativ: andere Funktion)
}

 

setTimeout

xxxxxxxxxx
setTimeout(() => {
    console.log("One second has elapsed.");
}, 1000);

setTimeout wird unmittelbar um die Funktion herum geschrieben, die verzögert werden soll.

setTimeout in for-loops

xxxxxxxxxx
for (let i = 0; i < n; i++) {       // let ist erforderlich
    setTimeout(() => {  
        // some code
    }, i * 1000);                   // i * Verzögerung
}

 

Strings

xxxxxxxxxx
let text = "Hello World";

Es spielt keine Rolle, ob einfache oder doppelte Anführungszeichen verwendet werden.

xxxxxxxxxx

.toUpperCase()

xxxxxxxxxx
"red".toUpperCase(); // "RED";

 

Einzelne Buchstaben eines Strings adressieren

xxxxxxxxxx
const language = "JavaScript";


language[0]; // "J", first character
language[1]; // "a", second character
// oder
language.at(-1); // "t", last character

 

.substring(x, y)

xxxxxxxxxx
const language = "JavaScript";
language.substring(1, 4); //"ava"

Die erste Zahl in der Klammer gibt die Position des ersten auszugebenden Buchstabens (beginnend bei 0!) an, die zweite Zahl nach dem Komma den ersten nicht mehr auszugebenden Buchstaben.

 

Strings mit dem Pluszeichen verbinden

xxxxxxxxxx
let prefix = "Mrs.";

+=

xxxxxxxxxx
name += " Blue"
// ist dasselbe wie
name = name + " Blue";

 

Template strings (kombinierte strings)

xxxxxxxxxx
let language = "JavaScript";
`I am learning ${language}`; //"I am learning JavaScript";

  • Template strings müssen von einem accent grave (backtick) umgeben sein.

  • Variablen werden mit $-Zeichen und geschweifter Klammer eingefasst.

  • Zeilenumbrüche sind möglich und werden bei der Ausgabe berücksichtigt.

 

.trim()

Entfernt Leerzeichen am Anfang und am Ende eines Strings.

 

.startsWith() und .endsWith()

xxxxxxxxxx
const sentence = "Hello there. Welcome!";


sentence.startsWith("Hello"); // true
sentence.startsWith("Sam"); // false


sentence.endsWith("Welcome!"); // true
sentence.endsWith("Welcome"); // false

 

.includes()

xxxxxxxxxx
const sentence = "Hello there. Welcome!";


sentence.includes("there"); // true
sentence.includes("Sam"); // false

 

.split(separator)

xxxxxxxxxx
let apps = "Calculator,Phone,Contacts";
let appsArray = apps.split(",");
console.log(appsArray); // ["Calculator", "Phone", "Contacts"]

 

.replace(search, replace) und .replaceAll(search, replace)

xxxxxxxxxx
const message = "You are welcome.";
message.replace(" ", "_"); // "You_are welcome"; (only the first occurrence has been replaced)


const message = "You are welcome.";
message.replaceAll(" ", "_"); // "You_are_welcome";


console.log(message); // "You are welcome" (original string is not changed)

  • Bei .replace() wird nur der erste Platzhalter ersetzt!

  • Der ursprüngliche String wird bei beiden Methoden nicht verändert.

 

Arrays

  • Arrays sind Varianblen, die mehrere Elemente enthalten (Zahlen, booleans, strings, objects oder gemischt).

  • Sie werden in eckige Klammern eingefasst.

xxxxxxxxxx
const values = [10, false, "John"];

 

Um ein einzelnes Element in einem Array zu adressieren, funktionieren zwei Schreibweisen:

xxxxxxxxxx
const users = ["Sam", "Alex", "Charley"];
users[1]; //"Alex"

oder:

xxxxxxxxxx
const users = ["Sam", "Alex", "Charley"];
users.at(-1); //"Charley"

 

Array-Methoden

.length

Zeigt die Länge (= Anzahl der Elemente) eines Arrays an

xxxxxxxxxx
const grades = [10, 8, 13, 15];
grades.length; // 4

 

.push

Element hinzfügen

xxxxxxxxxx
const numbers = [10, 8, 13, 15];
numbers.push(20); // returns 5 (the new length of the array)
console.log(numbers); // [10, 8, 13, 15, 20];

 

.forEach

Elemente durchschleifen (Iteration)

xxxxxxxxxx
let sum = 0;
grades.forEach(function(grade) {
    sum += grade
});

return from outside!

 

.filter

Gibt ein neues Array mit denjenigen Elementen aus, die die Bedingung erfüllen

xxxxxxxxxx
let numbers = [9, 5, 14, 3, 11];


let numbersAboveTen = numbers.filter(function(number) {
    return number > 10;
});
console.log(numbersAboveTen); // [14, 11]

return from inside!

 

Wird nach boolean gefiltert und es sollen nur Elemente mit der Eigenschaft true zurückgegeben werden, kann die Schreibweise mittels arrow function und implicit return folgendermaßen abgekürzt werden:

xxxxxxxxxx
const completed = data.filter(task => task.isCompleted);

 

.find

Gibt das erste Element zurück, das den Suchkriterien entspricht

xxxxxxxxxx
let names = ["Sam", "Alex", "Charlie"];


let result = names.find(function(name) {
  return name === "Alex";
});
console.log(result); // "Alex"

return from inside!

 

.every

Gibt 'true' aus, wenn alle Elemente die Bedingung erfüllen

xxxxxxxxxx
const numbers = [15, 10, 20];


const allAbove10 = numbers.every(number => number >= 10); // true
const allAbove15 = numbers.every(number => number >= 15); // false

 

.some

Gibt 'true' aus, wenn mindestens ein Element die Bedingung erfüllt

xxxxxxxxxx
const numbers = [15, 10, 20];


const someOver18 = numbers.some(number => number >= 18); // true
const someUnder10 = numbers.some(number => number < 10); // false

 

.map

Führt eine vorgegebene Operation mit jedem Element aus und gibt ein neues Array mit den Ergebnissen zurück

xxxxxxxxxx
const numbers = [4, 2, 5, 8];


const doubled = numbers.map(function(number) {
    return number * 2;
});
console.log(doubled); // [8, 4, 10, 16]

return from inside!

 

.join

Kann genutzt werden, um das Trennzeichen zwischen zwei Elementen eines Arrays vorzugeben. Z.B. wenn das Array als String an eine Website übergeben werden soll.

xxxxxxxxxx
const groceries = ["Apple", "Peach", "Tomato"];
groceries.join("; "); // "Apple; Peach; Tomato"
groceries.join(" . "); // "Apple . Peach . Tomato"
groceries.join(); // "ApplePeachTomato"

 

Array to String

Verbindet die Methoden .map und .join in einem Schritt

xxxxxxxxxx
const users = [{
    id: 1,
    name: "Sam Doe"
}, {
    id: 2,
    name: "Alex Blue"
}];


const html = `<ul>
    ${users.map(user => `<li>${user.name}</li>`).join("")}
    </ul>`;
console.log(html); // <ul> <li>Sam Doe</li><li>Alex Blue</li> </ul>

Hilfreich um html-Code auszugeben, da keine Kommata zwischen den einzelnen Elementen gesetzt werden!

 

.splice

Löscht Elemente aus einem Array

xxxxxxxxxx
const items = ["Pen", "Paper", "Staples"];
const deletedItem = items.splice(0, 1); // removes one element at index 0
console.log(deletedItem); // ["Pen"]


console.log(items); // ["Paper", "Staples"]

In der Klammer nach .splice wird als erstes angegeben, ab welcher Position gelöscht werden soll (beginnend bei 0!) und nach dem Komma die Anzahl der zu löschenden Elemente. Im obigen Beispiel würde items.splice(1, 1) nur "Paper" löschen.

 

.reduce

Reduziert mittels einer vorgegebenen Operation alle Elemente eines Arrays auf einen einzelnen Wert.

Beispiel Addition

xxxxxxxxxx
const grades = [10, 15, 5];


const sum = grades.reduce((total, current) => { 
    return total + current;
}, 0);
console.log(sum); // 30

 

Beispiel Multiplikation

xxxxxxxxxx
const numbers = [5, 2, 10];


const result = numbers.reduce((total, current) => {
    return total * current;
}, 1);
console.log(result); // 100

total und current müssen als 'Reducer' stehen bleiben. Nach der geschweiften Klammer und dem Komma steht das 'initialValue', das bei der Addition 0 und bei der Multiplikation 1 sein muss.

 

Arrays of objects

Syntax allgemein (für Methoden)

Wird bei einem Array aus objects ein bestimmter key benötigt, kann er mit folgender Schreibweise adressiert werden.

xxxxxxxxxx
tweets.filter(tweet => tweet.stats.likes > 30);

Hierbei wird die key-Hierarchie von tweets auf den Parameter innerhalb der Funktion (tweet) übertragen. Gilt analog für andere Methoden.

 

Syntax bei .reduce

xxxxxxxxxx
const sum = grades.reduce((total, current) => {
    return total + current.grade;
}, 0);

Hierbei wird die key-Hierarchie von grades auf den Parameter innerhalb der Funktion (grade) übertragen.

 

 

Objects

Syntax

xxxxxxxxxx
const user = {
    id: 1,
    firstName: "Sam",
    lastName: "Doe",
    age: 20
};


user.id; // 1
user.firstName; // "Sam"
user.isAdmin; // undefined (property does not exist)

  • Ein Objekt wird immer mit einer geschweiften Klammer eingefasst.

  • Es besteht aus Paaren von keys und values, die durch einen Doppelpunkt getrennt werden.

  • Paare werden durch Kommata getrennt.

  • Zum Auslesen einzelner keys wird dieser mit Punkt an die Objektvariable angefügt variable.key

 

Dynamic property - Keys als Variablen

xxxxxxxxxx
const user = {
    id: 1,
    name: "Sam Green",
};


const number = "id";
user[number]; // 1

Da in diesem Fall 'number' eine Variable ist, muss zum Auslesen die Schreibweise mit eckigen Klammern verwendet werden (s. letzte Zeile).

 

Object-Methoden

Object.keys()

Alle keys eines Objekts als Array ausgeben

xxxxxxxxxx
const user = {
    id: 1,
    name: "Sam Green",
    age: 20
};


const keys = Object.keys(user);
console.log(keys); // ["id", "name", "age"]

Object mit großem 'O'!

 

Object.values()

Alle values eines Objekts als Array ausgeben

xxxxxxxxxx
const user = {
    id: 1,
    name: "Sam Green",
    age: 20
};


const values = Object.values(user);
console.log(values); // [1, "Sam Green", 20]

 

Object.entries()

Alle key-value-Paare als Arrays ausgeben

xxxxxxxxxx
const user = {
    id: 1,
    name: "Sam Green",
    age: 20
};


const entries = Object.entries(user);
console.log(entries); //[["id", 1], ["name", "Sam Green"], ["age", 20]]

Die einzelnen Arrays werden in einem übergeordneten Array eingefasst.

 

Object shorthand

Wenn Variable und der dazugehörige key dieselbe Bezeichnung haben, kann folgende Syntax verwendet werden.

xxxxxxxxxx
const age = 18;
const person = {
    name: "John",
    age           // anstelle von 'age: age'
}

 

Analog kann bei console.log() die object-shorthand-Schreibweise verwendet werden, um immer sowohl den Namen der Variable (des Parameters) als auch ihren Wert auszugeben.

xxxxxxxxxx
const sum = (a, b) => {
    console.log({a}); // {a: 1}
    console.log({b}); // {b: 3}
    let total = a + b;
    console.log({total}); // {total: 4}
    return total;
}

 

Destructuring objects

Kurze Schreibweise, um Variablen aus Objekt-keys zu definieren

xxxxxxxxxx
// herkömmliche  Definition von Variablen aus einem Objekt 'config'
const id = config.id;
const isAdmin = config.isAdmin;
const theme = config.theme;


// destructuring Schreibweise
const {id, isAdmin, theme} = config;

  • Gleichzeitig können dabei auch Vorgabewerte für einzelne Variablen definiert werden, die jedoch nur angewendet werden, wenn der Wert nicht bereits vorher definiert war. Schreibweise: const {id = 1, isAdmin = true, theme = dark} = config;

  • Um in diesem Kontext eine Variable umzubenennen, benutzt man folgende Schreibweise: const {id: number, isAdmin: administrator, theme: colorScheme} = config;

 

Optional chaining

xxxxxxxxxx
const name = user.details?.name?.firstName;

  • Wird zur Vermeidung von Fehlermeldungen verwendet.

  • Ist im obigen Beispiel user.details und/oder user.details.name nicht definiert, wird 'undefined' ausgegeben.

 

Optional chaining in arrays

xxxxxxxxxx
const firstValue = data.temperatures?.[0];

  • Es wird 'undefined' zurückgegeben, wenn der erste Wert in data.temperatures leer oder undefiniert ist.

  • Beachte die Schreibweise ?.

 

Optional chaining bei Methoden

xxxxxxxxxx
const upperCasedName = person.name?.toUpperCase();

 

Nullish coalescing

Definiert einen Vorgabewert, falls nicht bereits definiert.

xxxxxxxxxx
const getName = name => {
    return name ?? "N/A";
}

oder in Verwendung mit einer Variablen

xxxxxxxxxx
const getWelcomeMessage = user => {
    return `Welcome ${user.fullName ?? "user"}`;
}

 

Klassen

Mit Klassen lassen sich Vorlagen erstellen, die Objekte und Funktionen enthalten und später immer wieder aufgerufen werden können.

xxxxxxxxxx
// class definition
class Person {
    constructor(firstName, lastName) {
        console.log(firstName + " " + lastName);
    }
}


// class usage
const person1 = new Person("Sam", "Green");
const person2 = new Person("Alex", "Blue");

  • Der Klassenname wird mit Großbuchstaben begonnen. Längere Bezeichnungen folgen diesem Schema: UpperCamelCase

  • Im constructor werden diejenigen Parameter aufgelistet, die später eingegeben werden und variieren können.

  • Um die Klasse (mitsamt ihren Funktionen) aufzurufen wird eine neue Variable (Bsp. person1) definiert und mit new Klassenname der Klasse zugewiesen (= Instanziierung).

 

Variablen innerhalb einer Klasse

xxxxxxxxxx
class Person {
    constructor(firstName, lastName) {
        // capture firstName param into this.firstName instance variable
        this.firstName = firstName;
        // capture lastName param into this.lastName instance variable
        this.lastName = lastName;
        // set an instance variable (without capturing a constructor param)
        this.votingAge = 18;
    }
}

Variablen innerhalb einer Klasse müssen mittels this.variableName = xxx einem Parameter oder einem Wert zugewiesen werden.

 

Methoden innerhalb von Klassen

xxxxxxxxxx
class Person {
    constructor(firstName, lastName) {
        this.firstName = firstName;
        this.lastName = lastName;
    }


    getFullName() {
        return `${this.firstName} ${this.lastName}`;
    }
    
    getGreeting() {
        const fullName = this.getFullName(); // call an instance method
        return `Hello ${fullName}`
    }
}


const person1 = new Person("Sam", "Green");
console.log(person1.getFullName()); // "Sam Green"

  • Methoden innerhalb von Klassen sind Funktionen (im Beispiel: getFullName()), benötigen aber keine Bezeichnung als function oder => arrow-Schreibweise.

  • Innerhalb einer Klasse können bereits vorhandene Methoden mit der this.method1 Schreibweise aufgerufen werden. (Im Beispiel wird so innerhalb der getGreeting Methode die getFullName Methode aufgerufen).

  • Einzelne Funktionen einer Klasse können mit der Dot-Schreibweise später außerhalb der Klasse separat aufgerufen werden (Bsp.: person1.getFullName())

static

Soll sich eine Methode innerhalb einer Klasse nicht auf die einzelnen später erzeugten Instanzen beziehen sondern immer angewendet werden, so kann sie mit static eingeleitet werden.

xxxxxxxxxx
class Config {
    static getYear() {

    }

Method chaining

Um mehrere Methoden (außerhalb der Klasse) nacheinander aufzurufen. können diese mit der dot-Schreibweise verbunden werden.

xxxxxxxxxx
course.markAsCompleted();
course.setGrade(18);
course.requestCertificate();
// wird zu:
course.markAsCompleted().setGrade(18).requestCertificate();

 

Voraussetzung: die betreffende Methode muss return this enthalten.

xxxxxxxxxx
class Course {
    constructor(name, isCompleted) {
        this.name = name;
        this.isCompleted = isCompleted;
    }


    markAsCompleted() {
        this.isCompleted = true;
        return this; // allows method chaining
    }


    setGrade(grade) {
        this.grade = grade;
        return this; // allows method chaining
    }


    requestCertificate() {
        this.askedForCertificate = true;
        return this; // allows method chaining
    }
}

 

Inheritance

Soll in einer Klasse all das angewendet werden was bereits zuvor für eine andere Klasse definiert wurde, kann dies mit dem Kommando extends aufgerufen werden.

xxxxxxxxxx
class Employee {
    constructor(firstName, lastName) {
        this.firstName = firstName;
        this.lastName = lastName;
    }


    getFullName() {
        return `${this.firstName} ${this.lastName}`;
    }


    getInitials() {
        return this.firstName[0] + this.lastName[0];
    }
}


class Manager extends Employee {
    someAdditionalMethod() {
        // ... some code
    }   
}

 

Soll eine der Funktionen in der erbende Klasse angepasst oder ergänzt werden, kann sie einfach neu geschrieben werden. (Es müssen keine neuen constructor-Parameter angegeben werden, solange sie identisch mit der vererbenden Klasse sind.)

xxxxxxxxxx
class Manager extends Employee {
    getFullName() {
        return `${this.firstName} ${this.lastName} (manager)`;
    }
}

 

Wird für die erbende Klasse eine zusätzliche Variable (im Beipiel userType) benötigt, müssen vorher mit super() die bestehenden Variablen abgerufen werden.

xxxxxxxxxx
class Admin extends User {
    constructor(firstName, lastName, age, userType) {
        super(firstName, lastName, age);
        this.userType = userType;
    }

 

Wird eine Methode aus der vererbende Klasse ergänzt, muss sie ebenfalls zunächst mit super.methode() aufgerufen werden. (Im Beispiel wird davon ausgegangen, dass getFullName() eine in der Klasse Manager bereits definierte Methode ist.)

xxxxxxxxxx
class Manager extends Employee {
    // ... some code


    getFullName() {
        return super.getFullName() + " [manager]";
    }
}

 

Public class fields

xxxxxxxxxx
class User {
    constructor() {
        this.votingAge = 18;
    }
}

Promise

Wird zur asynchronen Bearbeitung von Funktionen genutzt, d.h. dass Funktionen nacheinander abgearbeitet werden und aufeinander 'warten' bis bestimmte Bedingungen erfüllt oder nicht erfüllt sind.

 

new Promise

Um später eine Funktion so zu nutzen, dass erst abgewartet wird, bis eine bestimmte Bedingung erfüllt ist (oder nicht), muss sie return new Promise ((resolve) => { ... }); enthalten.

xxxxxxxxxx
const someFunction = () => {
    return new Promise((resolve) => {
        // do some work;
        resolve();
    });
}

Hinweis: Promise mit großem P!

 

resolve() kann als Parameter einen Wert (Variable, Zahl, Objekt, String) zugewiesen bekommen, welcher dann später beim Aufrufen als Ergebnis abgefragt werden kann (im Beispiel in .then(data))

xxxxxxxxxx
const wait = milliseconds => {
    return new Promise((resolve) => {
        setTimeout(() => {
            const seconds = milliseconds / 1000;
            resolve(seconds);
        }, milliseconds);
    });
}




// Funktion aufrufen
wait(2000).then(data => {
    console.log(data); // 2 (seconds)
});

 

Analog zu resolve kann ein zweiter Parameter namens reject in new Promise eingefügt werden, um zu definieren, was passiert, wenn z.B. resolve nicht erfolgreich ausgeführt werden kann. Typischerweise geschieht dies mit einer if-Kondition.

xxxxxxxxxx
new Promise((resolve, reject) => {
    if(...) {
       reject();
       }
    resolve();
});

 

.then

xxxxxxxxxx
someFunction().then(() => {
    console.log("executed after someFunction is completed");
});

Funktioniert nur im Zusammenhang mit new Promise in der ersten Funktion!

 

Wird von der ersten Funktion (sumTemperatures()) ein Ergebnis erwartet, das weiter verarbeitet werden soll, kann dieses Ergebnis als Variable (hier: value) der .then-Methode hinzugefügt werden. 

xxxxxxxxxx
sumTemperatures(temperatures).then(value => {
    console.log(value); // 18 (the sum of temperatures)
});

 

.catch

Funktioniert analog zu .then, wird aber nur dann ausgeführt, wenn .then nicht erfolgreich ausgeführt werden konnte. Typischerweise also für eine Fehlermeldung.

xxxxxxxxxx
getWeatherIn("Amsterdam")
    .then(data => {
        console.log(data);
    })
    .catch(error => {
        console.error(error); // {error: "Connection issue"}
    });

Wichtig: Kein Semikolon in Zeile 4!

 

.finally

Wird ausgeführt, wenn der Status von promise sich von pending zu fulfilled oder zu rejected ändert (also in beiden Fällen). Mit anderen Worten: .finally wird ausgeführt, egal ob .then erfolgreich war oder nicht. Syntax analog zu .then und .catch (kein Semikolon davor!).

xxxxxxxxxx
startLoader();
getWeatherIn("Amsterdam")
    .then(data => {
        console.log(data);
    })
    .catch(error => {
        console.error(error);
    })
    .finally(() => {
        stopLoader();
    });

fetch()

Daten von url holen. url in Anführungszeichen! fetch("https/ ... ")

Typisches Anwendungsbeispiel, um Daten von einer API zu holen:

xxxxxxxxxx
fetch("URL")
.then(response => response.json())
.then(data => {
    console.log(data);
    // b.B. weitere Verarbeitung der Daten hier
});

Sowohl fetch() als auch response.json() liefern Promises als Ergebnis. Daher muss die weitere Verarbeitung mit .then erfolgen. response und data sind frei benennbare Variablen, werden aber per Konvention so bezeichnet.

 

POST, PUT, DELETE

xxxxxxxxxx
fetch("URL", {
    method: "POST", // or PUT or DELETE
    headers: {
        "Content-Type": "application/json"
    },
    body: JSON.stringify({
        key1: 'value1', // replace with key/value based on documentation
        key2: 'value2', // same as above (if needed)
    })
})
.then(response => response.json())
.then(data => {
    console.log(data); // read server response
});

Um Daten zu senden (POST), zu ändern (PUT) oder zu löschen (DELETE), werden der fetch-Methode weitere Parameter (headers & body) hinzugefügt.

 

Umgang mit Fehlern

xxxxxxxxxx
fetch(URL)
    .then(response => {
        if (!response.ok) {
            // 4xx or 5xx error
            throw new Error("API issues.");
        }
        return response.json();
    })
    .then(data => {
        console.log(data);
    })
    .catch(error => {
        console.error(error);
    });

Der erste Teil ( if (!response.ok) { ... ) fängt zu 95% Fehler ab, die client-seitig (4xx) oder server-seitig (5xx) entstanden sind. In diesem Fall wurde fetch() jedoch erfolgreich durchgeführt, nur eben mit einer Fehlermeldung. throw new Error deklariert fetch() als rejected, somit greift dann auch .catch(), das sonst nur bei Netzwerkfehlern etc. greifen würde.

 

FetchWrapper

(FetchWrapper ist kein feststehender Ausdruck!). Mithilfe einer Klasse sollen diejenige Schritte automatisiert werden, die sonst bei jedem fetch-Vorgang manuell ausgeführt werden müssten. Typischerweise gliedert sich die URL in base-url und Endpoint, wobei meist nur der Endpoint variiert. Daher werden diese beiden Bestandteile in der FetchWrapper-Klasse getrennt verarbeitet.

xxxxxxxxxx
class FetchWrapper {
    constructor(baseURL) {
        this.baseURL = baseURL;
    }


    get(endpoint) {
        return fetch(this.baseURL + endpoint)
            .then(response => response.json());
    }


    put(endpoint, body) {
        return this._send("put", endpoint, body);
    }


    post(endpoint, body) {
        return this._send("post", endpoint, body);
    }


    delete(endpoint, body) {
        return this._send("delete", endpoint, body);
    }


    _send(method, endpoint, body) {
        return fetch(this.baseURL + endpoint, {
            method,
            headers: {
                "Content-Type": "application/json"
            },
            body: JSON.stringify(body)
        }).then(response => response.json());
    }
}

Zum Ausführen dann:

xxxxxxxxxx
const API = new FetchWrapper("https://base-url");


API.get("/endpoint.json").then(data => {
    console.log(data);
    // ... some more code
});

oder zum senden von Daten (PUT, POST, DELETE)

xxxxxxxxxx
const API = new FetchWrapper("https://base-url");


API.post("endpoint.json", {         // alternativ: API.put oder API.delete
    key_1: "value_1",
    key_2: "value_2"
    })
    .then(data => {
    console.log(data)
});

const paragraphs = document.querySelectorAll("p");

dataset

Mit Dataset können Datensätze aus der API gelesen oder an sie geschrieben werden. Konvention bei der Namensgebung von keys: data-someValue Mit diesmen Zusatz data- werden sie von dataset erkannt.

Auslesen

xxxxxxxxxx
<form id="payment-form" data-currency="EUR" data-user-id="2">
    ...
</form>
xxxxxxxxxx
const form = document.querySelector("#payment-form");

// Der Zusatz data- wird automatisch entfernt und die keys werden in camelCase umgewandelt!
const currency = form.dataset.currency; // "EUR"

Schreiben

xxxxxxxxxx
const navbar = document.querySelector("#navbar");
navbar.dataset.userId = 43;
navbar.dataset.rememberMe = false;

Cave! Alle Daten werden als Strings gespeichert. Beim Auslesen müssen sie also wieder zurückgewandelt werden.

xxxxxxxxxx
// Für Zahlen:
const userId = card.dataset.userId;
return Number.parseInt(userId, 10);


// Indirekter Weg, um Booleans abzugleichen:
const rememberMe = navbar.dataset.rememberMe === "true"; // false (!)

Bestehenden html-String einer bestimmten ID/Klasse ergänzen.

Beispiel eines typischen Formularfeldes:

    <input type="text" id="city" placeholder="Enter your city">

submit funktioniert analog zu click, jedoch nur im Zusammenhang mit forms.

removeEventListener()

Voraussetzung für das Entfernen eines EventListeners ist das Definieren einer Funktion, die beschreibt was entfernt werden soll. Diese Funktion muss zuvor mittels AddEventListener hinzugefügt worden sein.

xxxxxxxxxx
const button = document.querySelector("#my-button");
 
const handleClick = () => {                           // Funktion definieren
    console.log("button clicked");    
}


button.addEventListener("click", handleClick);        // Event hinzufügen


button.removeEventListener("click", handleClick);     // Event entfernen

Hinweis: Nach dem Funktionsnamen dürfen hier keine Klammern stehen, da die Funktion sonst immer ausgeführt würde, nicht erst beim Triggern.

Tipp! Wird ein Event vorhersehbar nur einmal benötigt, gibt es eine Lösung, die ohne removeEventListener auskommt. {once: true} entfernt den EventListener nach einmaligem Gebrauch automatisch.

xxxxxxxxxx
button.addEventListener("click", () => {
    console.log("button clicked");
}, {
    once: true
});

focus wird ausgelöst, wenn der Nutzer mit dem Cursor in den Fokus einer Textbox gelangt.

scroll wird jedesmal ausgelöst, wenn der Nutzer scrollt.

DOMContentLoaded wird ausgelöst, wenn der Browser das gesamte html gelesen hat (was nicht bedeutet, dass auch alle Inhalte bereits vollständig geladen wurden!).

    console.log("DOM is ready");

Lit

Lit ist eine library, die auf web components aufbaut und dessen Nutzung vereinfacht.

Zunächst muss die library installiert werden: npm install lit

xxxxxxxxxx
import {LitElement, html} from "lit";


class NewElement extends LitElement {
    render() {
        return html`<h1>Hello World!</h1>`;
    }
}


window.customElements.define("new-element", NewElement);

Im html-body:

xxxxxxxxxx
<new-element></new-element>
